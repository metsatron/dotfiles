;; Run one shell string via bash -lc <cmd>, print stdout, return exit code.
(define (sh cmd)
  (let* ((p   (open-pipe* OPEN_READ "bash" "-lc" cmd))
         (out (read-string p))
         (ec  (close-pipe p)))
    (display out)
    ec))

(define (task name desc thunk) (list name desc thunk))

(define (task-thunk t) (list-ref t 2))

(define (run name)
  (let ((t (find (lambda (t) (eq? (task-name t) name)) tasks)))
    (if t ((task-thunk t))
        (begin (format #t "Unknown task: ~a\n" name) 1))))

;; main: accept either `loom list` or `guile -s ... -- list`
(define (main args)
  (let* ((rest (cdr args))                    ; drop program name
         (rest (if (and (pair? rest)
                         (string=? (car rest) "--"))
                    (cdr rest) rest)))
    (match rest
      (("list")        (run 'list))
      (("help")        (run 'list))
      ((cmd . _more)   (run (string->symbol cmd)))
      (_               (begin
                         (format #t "Usage: loom <task>\nTry: loom list\n")
                         1)))))
#+END_SRC

ROOT   = $(PWD)

GUIX_HOME := $(HOME)/.config/guix
EXTRA := $(HOME)/.guix-extra-profiles

# Stow packages you normally manage here
STOW_PKGS ?= all linux debian think

.PHONY: toc tangle all guix-pull guix-core guix-dev guix-gc guix-dirs \
        stow safe-stow x11-apply bridge-flatpak bridge-flatpak-reset preview-stow

EMACS_BATCH = $(EMACS) --batch -Q \
  --eval "(setq create-lockfiles nil make-backup-files nil auto-save-default nil backup-inhibited t vc-make-backup-files nil)"

toc:
| $(EMACS_BATCH) --eval '(load-file "$(SCRIPT)")' --eval '(dotfiles-batch-update "$(ROOT)")'

tangle:
| $(EMACS_BATCH) --eval '(load-file "$(SCRIPT)")' --eval '(dotfiles-batch-tangle "$(ROOT)")'

# all = toc → tangle → guix-dirs only
all: toc tangle guix-dirs

# Plain stow
stow:
| cd $(HOME)/.dotfiles && stow $(STOW_PKGS)

# Safe stow with timestamped backups of real files
safe-stow:
| cd $(HOME)/.dotfiles && \
| for pkg in $(STOW_PKGS); do \
|   stow -n $$pkg 2>&1 \
|     | sed -n 's/.*\* existing target is neither a link nor a directory: \(.*\)$$/\1/p' \
|     | while read -r t; do \
|         if [ -f "$(HOME)/$$t" ] && [ ! -L "$(HOME)/$$t" ]; then \
|           cp -a "$(HOME)/$$t" "$(HOME)/$$t.bak.$$(date +%Y%m%d-%H%M%S)"; \
|         fi; \
|       done; \
|   stow $$pkg; \
| done

preview-stow:
| cd $(HOME)/.dotfiles && stow -n $(STOW_PKGS) || true

# X11 apply for think overlay
x11-apply: tangle
| cd $(HOME)/.dotfiles && stow think
| @echo "✅ X11 applied."

#+END_SRC

** Guix, profiles, and when targets run
- =make all= runs only =toc → tangle → guix-dirs=. It does not build profiles.
- Run =make guix-core= whenever you change =all/.config/guix/manifests/core.scm=.
- If you really want profiles built on every =make all=, add =guix-core= to the dependencies of =all=. This repo uses explicit targets to avoid surprise rebuilds.

** Session wiring
Linux only.

- =linux/.xsessionrc.d/20-guix-core-profile.sh= loads the named core profile so GUI apps launched from menus see Guix paths.
- =linux/.config/environment.d/20-gtk-appmenu.conf= exports GTK AppMenu variable session wide on Linux, safe across DEs.
- Make them live with:
#+BEGIN_SRC sh
cd ~/.dotfiles && stow all linux debian think
logout
#+END_SRC

** Remove or change specs in =core.scm=
- Re-run:
#+BEGIN_SRC sh
make guix-core
guix gc
#+END_SRC
- The profile reconciles with the manifest. Removed specs are unlinked from that profile. Store bytes free on =guix gc=.

| Target                 | What it does                                                                                       |
|------------------------+-----------------------------------------------------------------------------------------------------|

** What each powered feature actually does

*** Safe Stow (=safe-stow=)
- Runs =stow -n= on packages (=all linux debian think=) to **detect** files that aren’t symlinks yet.
- For each real file it would overwrite, copies a timestamped backup:
  =~/path/file.bak-YYYYmmdd-HHMMSS=
- Then runs =stow PKG= for each package in order.

Verify:
#+begin_src sh
readlink -f ~/.xsessionrc
ls -l ~/.local/bin/flatpak-desktop-bridge
#+end_src

Rollback (per file):
#+begin_src sh
mv ~/.xsessionrc.bak-YYYYmmdd-HHMMSS ~/.xsessionrc
#+end_src

*** X11 apply (=x11-apply=)
- Re-tangles (idempotent), stows =think=, and ensures your merged =~/.xsessionrc= contains:
  - your original GTK_CSD/LD_PRELOAD logic, and
  - a **non-clobbering** cursor search path:
    =XCURSOR_PATH=$HOME/.icons:/usr/local/share/icons:/usr/share/icons=

Verify:
#+begin_src sh
sed -n '1,80p' ~/.xsessionrc
printenv XCURSOR_PATH || echo "(will be set on next X session start)"
#+end_src

*** Flatpak bridge (fonts + cursors) (=bridge-flatpak=)
- Tangle+stow first (keeps scripts/configs current).
- Points all Flatpaks at a versioned fonts config:
  - =FONTCONFIG_FILE=${HOME}/.dotfiles/linux/.flatpak/fontconfig/fonts.conf=
  - That config includes:
    - your user fonts: =/home/metsatron/.local/share/fonts= (and legacy =~/.fonts= if used)
    - host fonts via the Flatpak bridge: =/run/host/usr/share/fonts= and =/run/host/usr/local/share/fonts=
- Exposes cursors to Flatpaks dynamically (no theme is forced):
  - =XCURSOR_PATH=$HOME/.icons:/run/host/usr/local/share/icons:/run/host/usr/share/icons=
- Unsets =LD_PRELOAD= inside Flatpaks to avoid host leakage.

Verify:
#+begin_src sh
flatpak override --user --show | grep -E 'FONTCONFIG_FILE|XCURSOR_PATH' || true
sed -n '1,40p' ~/.dotfiles/linux/.flatpak/fontconfig/fonts.conf

# Inside apps:
flatpak run --command=sh md.obsidian.Obsidian -c \
  'fc-cache -f >/dev/null 2>&1; fc-match "SFMono Nerd Font"; fc-match "SFMono Nerd Font Mono"'
flatpak run --command=sh one.ablaze.floorp -c 'fc-match "SFMono Nerd Font"'
flatpak run --command=sh one.ablaze.floorp -c 'echo $XCURSOR_PATH'
#+end_src

Expected:
- You see =SFMonoNerdFont-*.otf= matches in both apps.
- =XCURSOR_PATH= shows your user icons plus host icon dirs.

*** Flatpak bridge reset (=bridge-flatpak-reset=)
- Removes only the overrides the bridge set:
  - Unsets =FONTCONFIG_FILE= and =XCURSOR_PATH= at =--user= scope.
  - Removes any per-app env bindings added by the bridge.
- Does **not** touch your fonts or icons on disk.

Verify:
#+begin_src sh
flatpak override --user --show | grep -E 'FONTCONFIG_FILE|XCURSOR_PATH' || true
#+end_src

- =guix-core=: =tangle= → ensure =~/.guix-extra-profiles/core= → build from =$(GUIX_HOME)/manifests/core.scm=

- =guix-pull=: =guix pull=

** Troubleshooting

- Emacs “no version information available (libxml2)”
  This is benign on mixed Guix/Ubuntu setups; batch Emacs still runs. To silence it, align libxml2 versions or use a non-Guix Emacs for tangling.

- Fonts inside Flatpaks still resolve to DejaVu
#+begin_src sh
make bridge-flatpak
flatpak run --command=sh md.obsidian.Obsidian -c \
  'fc-cache -f >/dev/null 2>&1; FC_DEBUG=4 fc-match "SFMono Nerd Font"'
#+end_src
Check that =/home/metsatron/.local/share/fonts= is listed and =FONTCONFIG_FILE= points into =.dotfiles/linux/.flatpak/...=.

- Cursor theme not picked by some apps
Flatpaks see =XCURSOR_PATH=; the actual theme is chosen by the host. Ensure the selected theme exists in =~/.icons= or =/usr/share/icons=. We intentionally **don’t** set =XCURSOR_THEME= globally to keep it dynamic.

* Templates Directory

The `~/Templates/` directory provides structured starter files for use in Thunar's
"Create Document" context menu (or any other file manager that supports XDG Templates).

** Purpose
- Quick creation of new files with prefilled structure
- Cross-OS compatibility (even on systems without Thunar)
- Shared, stowable dotfiles-based templates

** Structure
We maintain a one-level-deep layout to avoid clutter in file manager menus:

#+BEGIN_EXAMPLE
Templates/
  Bash Script.sh
  Zsh Script.zsh
  Plain Text.txt
  README.md
  Obsidian Note.md
  Logseq Note.md
  Code/
  Emacs/
  Notes/
  Media/
  OpenDocument/
  Projects/
#+END_EXAMPLE

** Philosophy
- Keep most universal, OS-level, cross-project files in the *root*.
  - e.g. README.md, Plain Text.txt, Bash Script.sh
- Use subfolders for niche or domain-specific templates.
  - e.g. Code/, Emacs/, Notes/, Media/
- This preserves a clean, practical right-click menu.

** Usage
- Simply add or edit templates in ~/Templates/
- They will appear automatically in Thunar's "Create Document"
- Fully stow-compatible across systems

** Example Command to View Structure
Run from within Templates:

#+BEGIN_SRC bash
tree -a -L 2
#+END_SRC

This will show the top-level and immediate subfolder contents for easy verification.

** Notes
- Executable bits are not preserved when using "Create Document".
  - Scripts will need `chmod +x` after creation.
  - Templates contain correct shebangs for easy editing.
- Designed to be OS-agnostic.
  - Works on Linux (Thunar, Dolphin)
  - Safe to store on macOS, Windows, WSL
  - Can be used with CLI tooling
