#+TITLE: DotCortex Submodule Inventory
#+DESCRIPTION: Canonical ownership and update policy for git submodules
#+AUTHOR: Metsatron
#+LANGUAGE: en

* Policy
Submodule pointer updates are source-owned only when the path appears in this
inventory. Everything else stays operational-only until it gets an explicit
owner here.

Update flow:
- `loom git:submodules` may refresh the checkout for any listed path.
- Parent gitlink bumps may be committed only when the path is listed here.
- Nested submodule updates stay inside the child repo first, then the parent
  pointer is advanced.

* Notes
- This inventory is intentionally narrow: it covers the current managed
  submodule families without pretending every gitlink in the tree has the same
  policy.
- If a new submodule appears, add it here before advancing the parent pointer.
