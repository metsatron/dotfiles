* Backup entrypoints

These are the canonical backup commands and their targets, after
ironwolf01/ironwolf02 split.

** System mirror – ironwolf02

- Script :: backup-system
- Helper :: backup-system-clean
- Target root :: /mnt/ironwolf02/backup-xubuntu/backup-xubuntu
- Notes ::
  - backup-system-clean prunes/reset the mirror root safely.
  - backup-system rsyncs from / with your excludes and guard file:
    =.backup_guard= marking “system mirror target (rsync from /)”.

** Dotfiles snapshot – ironwolf02

- Script :: backup-dotfiles
- Source :: ~/DotCortex
- Target root :: /mnt/ironwolf02/backup-dotfiles
- Notes ::
  - One dated subdir per run (YYYY-MM-DD or whatever your script uses).
  - Requires /mnt/ironwolf02 mounted.

** Nextcloud mirror – ZFS → ironwolf02

- Script :: backup-nextcloud
- Live data :: /mnt/fideco-pool/Nextcloud   (ZFS: fideco-pool)
- Backup copy :: /mnt/ironwolf02/backup-nextcloud/Nextcloud
- Notes ::
  - Uses rsync from ZFS to ext4.
  - Sanity check ensures /mnt/ironwolf02 is mounted before writing.

** Games backup – ironwolf01

- Script :: backup-games
- Target root :: /backup/backup-games
- Disk :: ironwolf01 (mounted at /mnt/ironwolf01, with /backup -> /mnt/ironwolf01)
- Notes ::
  - Left exactly where it was before the new 12TB disk.
  - This is your Steam / GOG / etc. mirror.

** RetroPie backup – ironwolf01

- Script :: backup-retropie
- Target root :: /backup/backup-retropie
- Disk :: ironwolf01
- Notes ::
  - ROMs, configs, BIOS sets etc – same semantics as your existing script.

** Dedupe helper

- Script :: backup-dedupe
- Role :: Utility to trim duplicate trees across ironwolf01/ironwolf02/pool.
- Notes ::
  - No fixed target; run it explicitly against paths you choose.

verbs, so this is intentionally pseudo-code:

  - Run: =backup-games=
- backup:retropie
  - Run: =backup-retropie=

  - Run: =backup-dedupe <ARGS>=

Adjust the verb names and mappings to match how you’ve actually
