# Fresh-Context Verification

Use this when you need to verify completed DotCortex work from a fresh context.

## Steps

1. Re-read the changed files from a fresh context.
2. Run the smallest useful verification command.
3. Report the evidence and any remaining risk.

## Rules

- Verify from current state, not memory.
- Keep the report compact and factual.
